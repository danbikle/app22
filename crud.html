      I use the CRUD acronym as a start when I want to learn about a new API.
      The C in CRUD stands for Create.                                       
      The R stands for Retrieve.                                             
      The U stands for Update.                                               
      The D stands for Delete.                                               
      After I can use a given API to do CRUD operations on objects,          
      then I am on a path towards understanding that API.

When I do CRUD with D3.js I operate on two types of things.

First I CRUD HTML-elements like div-elements and svg-elements. Also I CRUD JavaScript objects related to the D3.js API.

HTML CRUD:

  • Create Method 1: Type it in via emacs
  • Create Method 2: D3 call to append()
  • Retrieve: Just Click Reload Button
  • Update Methods: D3 calls: append(), attr(), style(), html(), text()
  • Delete Methods: D3 call: remove()

D3.js CRUD:

  • Create Method 1: obj1 = d3.select('ul#mylist');
  • Create Method 2: obj2 = obj1.selectAll('li');
  • Retrieve: Just list the object name (useful in debugger-console): obj2;
  • Update Methods 1: D3 calls: append(), attr(), style(), html(), text()
  • Update Method 2: D3 call: obj2.data(myarray);
  • Create Method 3 (left over data into obj3): obj3 = obj2.enter();
  • Create Method 4 (left over HTML into obj4): obj4 = obj2.exit();
  • Delete Methods: D3 call: obj4.remove()
